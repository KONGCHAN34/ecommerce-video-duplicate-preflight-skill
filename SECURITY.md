# Security Policy

## Reporting Security Issues

Please do not open public issues for sensitive security reports.

Security-sensitive reports include:

- Input paths or filenames that expose private customer, platform, or account data.
- Reports containing private video material or unpublished campaign details.
- Vulnerabilities that could cause unintended file deletion, overwrite, upload, or disclosure.
- Credential, token, cookie, API key, or login-state exposure.
- Unsafe command execution, path traversal, or report injection behavior.

For now, report sensitive issues privately to the maintainer through the contact method listed on the maintainer's GitHub profile.

## Maintainer Response

The maintainer should:

1. Acknowledge the report.
2. Reproduce with sanitized fixtures where possible.
3. Patch without exposing private user material.
4. Credit the reporter only with explicit permission.
5. Publish a security note when the issue affects public users.
