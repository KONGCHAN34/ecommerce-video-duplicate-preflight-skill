# 30 Day Plan

## Days 1-3: Public v0.1.0

- Initialize the GitHub repository.
- Publish README, Skill, scanner, examples, governance files, and validation workflow.
- Create `v0.1.0` release.
- Open initial backlog issues.

## Days 4-10: Dogfood

- Run the scanner against three sanitized or non-private video batches.
- Record what the report got right and where the wording was unclear.
- Improve report fields and examples.
- Release `v0.1.1` if fixes are small, or `v0.2.0` if frame sampling begins.

## Days 11-20: Feedback

- Ask 2-5 content, editing, or e-commerce operators to review the workflow.
- Collect feedback as GitHub issues without private media.
- Improve installation notes and report language.
- Add one platform-neutral editor review example.

## Days 21-30: Strengthen Evidence

- Prepare release notes and a maintenance summary.
- Confirm GitHub profile and repository are public.
- Prepare Codex for Open Source notes honestly.
- Decide whether to apply immediately or wait for more adoption evidence.
